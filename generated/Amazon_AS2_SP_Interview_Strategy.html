⑧ 技术面 — 最可能考的方向 & 准备策略
ML System Design（最重要，必考）

典型题："Design a CTR/CVR prediction system for sponsored search at Amazon scale."

答题框架：
  • Problem framing：定义 objective（点击率？转化率？ROAS？），明确 constraint（latency <10ms，real-time serving）
  • Data：Features（query, product, advertiser, user context, historical CTR），label construction（delayed feedback 怎么处理），data freshness
  • Model：Two-tower / DSSM for retrieval，GBDT/DNN for ranking，calibration layer
  • Multi-objective：如何 balance CTR vs 广告主 bid vs relevance（Pareto frontier，Lagrangian relaxation）
  • Serving：Online serving latency，feature store，model update frequency
  • Evaluation：Offline (AUC, calibration) vs online (A/B test，revenue, RPM)
你的加分：ROKT 真实 RTB 经验，可以说"在 ROKT 我们面临类似的问题，我们的解法是..."
Auction Theory & Bidding（必须懂基本概念）
必知概念：
  • Second-price auction (GSP)：Amazon SP 的基础机制，赢家付第二名出价
  • Quality Score：Amazon 的 ad rank = bid × predicted CTR × relevance score
  • eCPM (effective CPM)：如何比较不同出价格式（CPC, CPM, ROAS）
  • Budget pacing：日预算如何平滑分配，避免 early exhaustion
  • Bid landscape：如何估计赢得某次竞拍需要的最低出价
你的加分：ROKT Strategy 7 事故直接展示你对 quality score 机制的深度理解
Coding（不用完美，但要干净）

Amazon AS-II coding bar：LeetCode medium。重点考数据结构和算法清晰度，不是 hard DP。

高频题型：
  • Hash map / two-pointer（频率统计，sliding window）
  • Binary search（bid threshold，operating point search）
  • Tree traversal（广告类目层级）
  • 偶尔：基础 graph（广告主-商品关系）
Python 优先。写之前先说思路，写完说 time/space complexity，问"要不要 handle edge cases？"
